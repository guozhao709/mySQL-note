# DDL

## SQL命名规范

1. 数据库名、表名不得超过30个字符，变量名限制为29个
2. 使用字母数字下划线命名，不能数字开头
3. 在sql语句中，如果要是用关键字来命名，需使用反引号扩起来

开发约定：

1. 一般使用下划线命名法
2. 库名应与对应的程序名一致

# 库

## 创建数据库

```sql
/*
1. 创建数据库
CREATE DATABASE IF NOT EXISTS <数据库名>;
2. 创建数据库并指定字符集和排序方式
CREATE DATABASE IF NOT EXISTS <数据库名> CHARACTER SET <字符集> COLLATE <排序方式>;
MYSQL默认字符集为utf8mb4, 排序方式为utf8mb4_0900_ai_ci(大小写不敏感), utf8mb4_0900_as_cs(大小写敏感)
*/

CREATE DATABASE IF NOT EXISTS DDL_DB CHARACTER SET utf8mb4 COLLATE utf8_0900_as_cs;
```

## 库查看和切换

```sql
/*
1. 查看所有库
SHOW DATABASES;
2. 查看当前使用库
SELECT DATABASE();
3. 查看库下所有表
SHOW TABLES FROM <>数据库名
4. 查看创建库的信息和语句
SHOW CREATE DATABASE <数据库名>
5. 选中和切换库
USE <数据库名>
*/

SHOW DATABASES;
SELECT DATABASE();
SHOW TABLES FROM mysql;
SHOW CREATE DATABASE people_db;
```

## 库的修改和删除

```sql
/*
1. 修改字符集和排序方式
ALTER DATABASE <数据库名> CHARACTER SET <字符集> COLLATE <排序方式>;
2. 删除数据库
DROP DATABASE IF EXISTS <数据库名>;
*/

ALTER DATABASE DDL_DB CHARACTER SET UTF8MB4 COLLATE UTF8_0900_AI_CI;
DROP DATABASE IF EXISTS DDL_DB;
```

练习
```
场景1:
假设你正在为一个多语言的博客平台设计数据库。你需要创建一个名为 blog_platform 的数据库,
支持存储多语言的文章和评论。由于博客平台可能包含来自不同语言的用户,你决定使用utf8mb4字符集,
排序方式选择默认值,以支持广泛的 Unicode 字符
场景2:
查看数据库字符集和排序规则
场景3:
假设在后续的发展中,你决定将排序方式修改为 utf8mb4_0900_as_cs,以实现大小写敏感的比较。
场景4:
查看数据库排序规则
场景5:
项目惨遭放弃,需要删除项目库,并且跑路
```

```sql
CREATE DATABASE IF NOT EXISTS blog_platform CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
USE blog_platform; 
SHOW VARIABLES LIKE 'character_set_database';

ALTER DATABASE blog_platform COLLATE utf8mb4_0900_as_cs;

```

# 表

## 创建表

```sql
CREATE TABLE IF NOT EXISTS <表名>(
    <列名> <数据类型> [约束条件] [COMMENT'注释'],
    <列名> <数据类型> [约束条件] [COMMENT'注释'],
) [表约束条件]
```
注:
1. 表名、列名、类型必须填写
2. 列与列之间, 使用逗号分割

## 列的类型

### 整数类型

- 标准 sql:

1. int: 4字节
2. smallint: 2字节

- mysql:

1. tinyint: 1字节
2. mediumint: 3字节
3. bigint: 8字节

在类型后面加上 unsigned, 表示无符号, 若为负数, 则转为其绝对值

### 浮点类型

1. float(m, d): 4字节, m: 24, d: 8
2. double(m, d): 8字节, m: 53, d: 30

其中m表示整个数的总位数, d表示小数的位数, 超出的位数会进行四舍五入

### 定点类型

decimal(m, d): 动态长度, m: 65, d: 30

用十进制存储, 无精度问题

### 短字符型

1. char(M): 最多255个字符, 固定长度(指定的M), 若不足, 末尾会用空格填充, 读取时会自动去掉空格
2. varchar(M): 最多65535个字节(M最大为16383), 不定长度, 实际为多少, 占用就为实际+1字节(因为要有1为表示空), 会保留末尾的空格
3. text: 大文本类型, 不占有一行最大字节数的限制, 自己最大为65535字节
4. LongText: 最大为4294967295字节(4GB)
5. tinytext: 最多255个字节
6. mediumtext: 最多16777215个字节(16MB)

但不建议使用大文本, 可以存储其地址, 使用文件系统

### 日期类型

1. year: yyyy
2. time: hh:mm:ss
3. date: yy-mm-dd
4. datetime: yy-mm-dd hh:mm:ss(1000-9999)
5. timestamp: yy-mm-dd hh:mm:ss(1970-2038)

```sql
CREATE TABLE IF NOT EXISTS T2(
    stu_name VARCHAR(20),
    register_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
)

```

## 表和列的修改与删除

1. 添加列
```sql
ALTER TABLE <表名> ADD <列名> <数据类型> [first | after 列名];
```
2. 修改列名
```sql
ALTER TABLE <表名> CHANGE <旧列名> <新列名> <数据类型> [first | after 列名];
```
3. 修改列的类型
```sql
ALTER TABLE <表名> MODIFY <列名> <数据类型> [first | after 列名];
```
4. 删除列
```sql
ALTER TABLE <表名> DROP <列名>;
```
5. 修改表名
```sql
ALTER TABLE <表名> RENAME <新表名>;
```
6. 删除表
```sql
DROP TABLE IF EXISTS <表名>;
```
7. 清空表数据
```sql
TURNCATE TABLE <表名>
```
