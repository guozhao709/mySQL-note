# 关系

## 一对一关系

一般用父表的主键当作子表的外键, 同时子表的主键一般也是外键

```
constraint a_a_fk foreign key(a_id) references a_table(a_id)
```

## 一对多关系

正常创建主子表, 添加外键约束即可

```sql
constraint a_b_fk foreign key(a_id) references a_table(a_id)
```

## 多对多关系

需要通过一个中间表来实现多对多关系

# 多表查询

分为两种: 垂直合并语法 和 水平合并查询(连接)

## 垂直合并

列数和对应列的类型相同即可, 会以写在上面的列名为新虚拟表的列名

前者去除重复行, 后者不去除

```sql
select * from a_table
union [all]
select * from b_table;
```

## 水平合并类型

要求: 两个表必须要有关系(主外键), 主外键相等

### 内连接

标准语法:

```sql
SELECT * FROM a_table
[INNER] JOIN b_table
ON a_table.主键 = b_table.外键
```

方言:

```sql
SELECT * FROM a_table, b_table
WHERE a_table.主键 = b_table.外键
```

### 外连接

分为左外连接和右外连接

```sql
# 左连接
SELECT * FROM table1 t1 LEFT [OUTER] JOIN table2 t2 ON  t1.p_key = t2.f_key; 
# 右连接
SELECT * FROM table1 t1 RIGHT [OUTER] JOIN table2 t2 ON  t1.p_key = t2.f_key; 
```

### 自然连接(不推荐使用)

可以自动的找到主外键相等的列, 因语法繁琐, 所以不推荐使用

```sql
SELECT * FROM table1 t1 NATURAL JOIN table2 t2 USING(t2.tid); 
```

### 自连接(用法)

单张表多次使用的一种场景

比如, 在一个员工表中有 eid, ename, mid(领导id)。这时要查询 eid, ename, mid(领导id), 领导姓名; 这时要使用自连接。

```sql
SELECT e1.eid, e1.ename, e1.mid, e2.ename AS leader_name
FROM t_employee e1 LEFT JOIN t_employee e2
ON e1.mid = e2.eid;
```

### 嵌套查询

在sql语句中嵌套一个完整的select查询语句

```sql
SELECT stu_id, stu_name, score FROM t_score
WHERE score > (
    SELECT AVG(score) FROM t_score
)
```

子查询有四种返回结果:

1. 单行单列: 一般用于条件判断
2. 单行多列: 通常用于更新和插入数据
3. 多行单列: 通常用于 IN, ANY, ALL 等条件语句中
4. 多行多列: 不用与查询条件, 用于连接的虚拟表

#### 在update和delete中使用嵌套查询

当两个语句同时引用一张表时, 不能进行修改, 但可以用如下的方式进行更新数据

```sql
UPDATE t_employee SET salary = (SELECT salary FROM (SELECT salary FROM t_employee WHERE ename = 'Blays') AS temp) WHERE ename = 'Bob';
```

#### 在insert中使用嵌套查询

```sql
# 创建一个表, 并复制一个表结构
CREATE TABLE table2 LIKE table1;

# 复制数据
INSERT INTO table2 (SELECT * FROM table1);

# 创建表时并复制数据
CREATE TABLE table3 AS (SELECT * FROM table1);
```
