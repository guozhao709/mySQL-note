# 一对一关系

一般用父表的主键当作子表的外键, 同时子表的主键一般也是外键

```
constraint a_a_fk foreign key(a_id) references a_table(a_id)
```

# 一对多关系

正常创建主子表, 添加外键约束即可

```
constraint a_b_fk foreign key(a_id) references a_table(a_id)
```

# 多表查询

分为两种: 垂直合并语法 和 水平合并查询(连接)

## 垂直合并

列数和对应列的类型相同即可, 会以写在上面的列名为新虚拟表的列名

前者去除重复行, 后者不去除
```
select * from a_table
union [all]
select * from b_table;
```
