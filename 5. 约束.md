# 约束

## 约束分类

- 域(列)级约束

1. 非空约束: NOT NULL, 列非空约束
2. 默认值约束: DEFAULT 某列默认值
3. 检查约束: CHECK

- 实体(行)级约束

1. 主键约束: PRIMARY KEY, 唯一标识且不为空
2. 外键约束: UNIQUE, 唯一标识, 但可以多个为空
3. 自增长约束: AUTO_INCREMENT, 自动递增, 且唯一标识

- 引用(多表)级约束

1. 外键约束: FOREIGN KEY 限定表中某一列, 正确引用其他表的数据值

格式:

```
CREATE TABLE 表名 (
    列名 数据类型 约束,
    ...,
);
```

```sql
CREATE TABLE IF NOT EXISTS student(
id INT PRIMARY KEY AUTO_INCREMENT,
name VARCHAR(20) NOT NULL,
);

```

检查约束比较特殊, 它属于表级别的约束

```sql
CREATE TABLE IF NOT EXISTS e_table (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(20) NOT NULL,
    age INT,
    CHECK (age >= 0 AND age <= 120),
)

ALTER TABLE e_table
ADD CONSTRAINT age_check CHECK (age >= 0 AND age <= 120);

ALTER TABLE e_table
DROP CONSTRAINT age_check;
```

所以它与列是平级的, 但可以约束列的数据

### 唯一约束

关键字: UNIQUE

特点:

1. 同一个表可以有多个唯一约束
2. 唯一约束允许列值为空
3. 在创建唯一约束的时候, 如果不给唯一约束命名, 就默认和列名同名

单值唯一约束: 只约束单列的唯一性

```sql
CREATE TABLE IF NOT EXISTS e_table (
   字段名 数据类型 UNIQUE,
)
```

多值组合唯一约束

```sql
CREATE TABLE IF NOT EXISTS e_table(
    CONSTRAINT 约束名 UNIQUE (列名1, 列名2, ...),
    ...,
)
```

建表后修改

```sql
ALTER TABLE e_table
ADD CONSTRAINT 约束名 UNIQUE (列名1);
```

删除上面有例子

### 主键约束

确保表中至少有一列的数据不重复

主键可以是多个列的组合, 确保这些列的组合不重复

一个表有且只有一个主键约束

建表时添加

```sql
CREATE TABLE IF NOT EXISTS e_table (
    id INT PRIMARY KEY,
)

CREATE TABLE IF NOT EXISTS e_table (
    id INT,
    name VARCHAR(20) NOT NULL,
    CONSTRAINT pk_id PRIMARY KEY (id),
)
```

建表后修改

```sql
ALTER TABLE e_table
ADD PRIMARY KEY (id);
```

删除

```sql
ALTER TABLE e_table
DROP PRIMARY KEY;
```

### 自增长约束

关键字: AUTO_INCREMENT

只能增加到主键列上, 给自增长字段设置0或者null, 自动递增, 否则设置真实值

自增长每次都从最大值开始递增

### 外键约束

添加外键约束

```sql
CREATE TABLE IF NOT EXISTS ee_table (
    eid INT,
    name VARCHAR(20) NOT NULL,
    CONSTRAINT fk_id FOREIGN KEY (id) REFERENCES e_table(id),
)
```

```SQL
# 先删除约束
ALTER TABLE ee_table
DROP FOREIGN KEY fk_id;
# 再删除外键索引
SHOW INDEX FROM ee_table;
ALTER TABLE ee_table
DROP INDEX fk_id;
```

追加外键约束

```sql
ALTER TABLE ee_table
ADD  CONSTRAINT fk_id FOREIGN KEY (id) REFERENCES e_table(id);
```

外键级联操作

当删除主表中的数据时, 如果子表有引用主表数据, 则会报错

```
CASCADE: 当父表发生操作时, 自动操作子表中的数据
SET NULL: 当父表发生操作时, 子表中的数据设为NULL
NO ACTION: 不执行任何操作, 但会检查, 如果删除主表数据时, 会抛异常
RESTRICT: (默认), 同NO ACTION
SET DEFAULT: 当父表发生操作时, 子表中的数据设为默认值
```

最好采用 ON UPDATE CASCADE ON DELETE CASCADE

eg:

```sql
CREATE TABLE IF NOT EXISTS ee_table (
    eid INT,
    name VARCHAR(20) NOT NULL,
    CONSTRAINT fk_id FOREIGN KEY (id) REFERENCES e_table(id) ON UPDATE CASCADE ON DELETE CASCADE,
)
```
